# Paste ini sebagai instruksi awal ke Claude Code saat mulai coding Baca semua file di folder design_handoff_sakuwise/ sebelum mulai coding. Urutan baca: 1. README.md — gambaran menyeluruh semua token, komponen, screen, arsitektur 2. Sakuwise Prototype.html — buka di browser, eksplorasi semua layar + dark mode 3. Sakuwise Handoff Spec.md — token → Compose mapping konkret 4. Sakuwise Technical Solution v1.1.md — arsitektur, module structure, stack detail 5. proto/components.jsx — spek komponen yang akan di-port ke :core:designsystem 6. proto/icons.jsx — semua 45+ ikon SVG (port ke VectorDrawable atau ImageVector) Prinsip implementasi: - App ID: com.gustiadhitya.sakuwise - Target: Android Kotlin 2.0 + Jetpack Compose + Material 3 + Hilt + Room + SQLCipher - Prototype HTML adalah spec visual final. Kalau konflik antara dokumen dan prototype, PROTOTYPE YANG MENANG. - Semua nominal Rupiah WAJIB fontFeatureSettings = "tnum" — gunakan RupiahText composable dari :core:ui. JANGAN format Rupiah tanpa tnum. - Gunakan sp (bukan dp) untuk semua text size. - ViewModel hanya inject UseCase, tidak pernah Repository langsung. - TIDAK ADA permission INTERNET di AndroidManifest.xml V1. - Semua 13 A11Y action items wajib check-off sebelum release ke Play Store. - Clean Architecture multi-module: ~13 modul Gradle sesuai README §3. - Strict UseCase pattern: setiap operasi data punya UseCase eksplisit. - Feature module TIDAK boleh depend ke feature lain atau ke :core:data/:core:database. Mulai dari: implementasi :core:designsystem terlebih dahulu (SakuwiseTheme, semua token, SW* components), lalu :feature:onboarding, lalu :feature:dashboard.
